# Form Client Components

Components specifically for form inputs, validation, and submission.

## Purpose

- Handle form input and validation
- Manage form state and submission
- Provide user feedback during form interaction
- Handle client-side form validation

## Examples

- Input fields with validation
- Form submission handlers
- File upload components
- Multi-step form wizards
- Form validation feedback

## Current Components to Migrate

Consider moving these existing form components here:

- `input.tsx` - Text input component
- `checkbox.tsx` - Checkbox input
- `radio-group.tsx` - Radio button group
- `select.tsx` - Select dropdown
- `switch.tsx` - Toggle switch
- `slider.tsx` - Range slider
- `one-time-password-field.tsx` - OTP input
- `password-toggle-field.tsx` - Password field

## Guidelines

- Always include `'use client'` directive
- Handle form state and validation
- Provide immediate user feedback
- Support accessibility features
- Follow form best practices
