# Interactive Client Components

Components that handle user interactions and browser events.

## Purpose

- Handle user clicks, hovers, and other interactions
- Manage component-level state
- Provide immediate user feedback
- Handle browser-specific functionality

## Examples

- Buttons with click handlers
- Modals and dialogs
- Dropdown menus
- Interactive forms
- Tooltips and popovers
- Tabs and accordions

## Guidelines

- Always include `'use client'` directive
- Handle events and user interactions
- Manage local component state
- Provide accessibility features
- Follow constitution validation rules
