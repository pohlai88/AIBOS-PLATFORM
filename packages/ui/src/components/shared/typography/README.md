# Typography Shared Components

Text and content components that work in both Server and Client environments.

## Purpose

- Provide consistent text styling
- Ensure typography hierarchy
- Support accessibility requirements
- Work in both server and client contexts

## Examples

- Headings (h1, h2, h3, etc.)
- Paragraph text
- Code blocks
- Lists and formatted text
- Text emphasis and styling

## Guidelines

- Use semantic HTML elements
- Follow typography scale from design tokens
- Support WCAG accessibility requirements
- Accept text content as children
- No client-specific functionality
