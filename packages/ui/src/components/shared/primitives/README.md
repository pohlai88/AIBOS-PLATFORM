# Primitive Shared Components

Basic UI building blocks that work in both Server and Client environments.

## Purpose

- Provide fundamental UI elements
- Ensure consistent styling across the application
- Work seamlessly in both server and client contexts
- Serve as building blocks for more complex components

## Examples

- Layout containers and wrappers
- Basic styling components
- Spacing and alignment utilities
- Simple display components

## Guidelines

- No client-specific code
- Accept styling props and className
- Use semantic HTML elements
- Follow accessibility best practices
- Support both server and client rendering
